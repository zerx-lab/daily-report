消息接收通道

推荐 · 默认 HTTP POST 事件上报

NapCat 将消息事件通过 HTTP POST 发送到本系统。复用现有 Web 端口，无需额外配置，始终可用。

NapCat 侧配置步骤：

  1. 打开 NapCat WebUI → 网络配置
  2. 添加 HTTP 上报 连接
  3. 上报地址填写：http://本机IP:{{if .bot}}{{if index .bot "ws_port"}}{{end}}{{end}}当前端口/onebot/v11/http
  4. 如有 Access Token，填写与上方一致的值
  5. 保存并启用
可选 反向 WebSocket

NapCat 主动建立 WebSocket 长连接。需要额外监听端口，适合追求更低延迟或同时使用双通道的场景。不启用不影响机器人正常工作。

NapCat 侧配置反向 WS 地址为：ws://本机IP:{{if .bot}}{{index .bot "ws_port"}}{{end}}/onebot/v11/ws
NapCat 快速配置指引
最少步骤让机器人跑起来

本系统侧（当前页面）

  1. 勾选 启用 QQ 机器人
  2. 填写 NapCat HTTP API 地址（NapCat 运行地址）
  3. 填写 QQ 号白名单（你的 QQ 号）
  4. 保存并重启服务

NapCat 侧（NapCat WebUI）

  1. 启用 HTTP 上报
  2. 上报地址填本系统的 /onebot/v11/http
  3. 启用 HTTP 服务（机器人回消息用）
  4. 保存，向机器人发消息测试
只需 HTTP 上报即可工作，反向 WebSocket 是可选的。两者可同时启用互为备份。 消息发送始终通过 NapCat 的 HTTP API 地址。
